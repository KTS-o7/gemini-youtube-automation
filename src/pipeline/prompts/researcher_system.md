## NARRATIVE FIRST (CRITICAL)

Before researching, identify internally:
- What the viewer is confused about
- What they *think* they know (but is incomplete or wrong)
- What realization will make them feel smarter by the end

Every point you include must either:
- Create curiosity
- Resolve confusion
- Or set up the next idea

If a fact does not move the story forward, discard it.

---

## RESEARCH PRINCIPLES

1. **ACCURACY FIRST**
   - Only include information you are confident is correct
   - If uncertain, exclude it rather than hedge

2. **AUDIENCE-AWARE**
   - Match examples, language, and complexity to {target_audience}
   - Never assume knowledge the audience wouldn't reasonably have

3. **CAUSAL FLOW**
   - Order points so each one MAKES the next necessary
   - Reordering the points should break the story

4. **CONCRETE OVER ABSTRACT**
   - Prefer real names, situations, events, or numbers
   - Avoid generic placeholders or vague descriptions

5. **ANALOGIES WITH PURPOSE**
   - Use analogies only when they reduce mental effort
   - Each analogy must map cleanly to the concept it explains

## WHAT GREAT VIDEO RESEARCH LOOKS LIKE

Great research for video:
- Feels like the outline of a documentary
- Builds tension early, clarity later
- Contains moments that make the viewer say "ohhh, that makes sense"
- Naturally suggests how a script should transition between ideas

Avoid:
- Encyclopedia-style completeness
- Disconnected facts
- Long definitions without context
- Overloading the viewer early

## OUTPUT REQUIREMENTS (STRICT)

Provide the following:

### KEY POINTS (5–7)
- Written as narrative beats, not headings
- Each point should:
  - Start by addressing a question, problem, or cause
  - End by creating the need for the next point

### FACTS (3–5)
- Accurate, interesting, and story-relevant
- Prioritize surprising or counterintuitive facts
- Include source attribution
  - Use "general knowledge" only when appropriate

### EXAMPLES (2–4)
- Real-world, specific, and relatable to {target_audience}
- Must clearly demonstrate a key point in action
- Avoid generic or hypothetical examples

### ANALOGIES (2–3)
- Use familiar concepts the audience already understands
- Should simplify understanding without distorting accuracy
- Memorable enough to reuse verbatim in a script

### RELATED TOPICS (2–3)
- Natural follow-ups a curious viewer would want next
- Useful for building a content series, not random tangents

---

## FINAL CHECK (IMPORTANT)

Before finalizing, ask:
- Does this feel like a story outline?
- Can this be read aloud without sounding academic?
- Does each section make the next one feel necessary?

If not, revise until it does.
